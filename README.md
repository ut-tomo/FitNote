# Diet Tracker

Rust 製のローカルデスクトップ食事・体重管理アプリです。
インターネット接続不要でデータをすべてローカルの SQLite に保存し、AI コーチ機能（Google Gemini API）でオプションの週次レポートを生成できます。

---

## スクリーンショット / 画面構成

```
┌──────────────────────────────────────┐
│  Diet Tracker          [トースト通知]  │  ← トップバー
├──────┬──────┬──────┬────────────────┤
│ 今日 │ 食品 │グラフ│ レポート        │  ← タブバー
├──────────────────────────────────────┤
│                                      │
│          メインコンテンツ             │
│                                      │
└──────────────────────────────────────┘
```

4 つのタブで機能が分かれています。

---

## 機能概要

### 今日タブ (`src/ui/home.rs`)
今日の食事と体重を記録する中心的な画面。

| 機能 | 説明 |
|---|---|
| 体重記録 | kg を入力して「保存」ボタンまたは Enter キーで保存 |
| 栄養サマリ | kcal / P / F / C のプログレスバーで目標比を可視化 |
| 食事記録 | 朝食・昼食・夕食・間食のスロットを選択し、食品をインクリメンタル検索して量を入力し追加 |
| 食品ログ | スロット別に記録済み食品一覧を表示。「×」ボタンで個別削除可能 |

- カロリーが目標の **90% 未満** → グレー表示
- カロリーが目標の **90〜110%** → オレンジ（達成）
- カロリーが目標の **110% 超** → 赤（超過）

---

### 食品タブ (`src/ui/foods.rs`)
食品マスタを自由に管理する画面。食品を登録しておくと「今日」タブの検索に使えます。

| 機能 | 説明 |
|---|---|
| 食品追加 | 名前・単位・1単位あたりの kcal / P / F / C を入力して登録 |
| 食品編集 | 一覧の「編集」ボタンをクリックするとフォームが編集モードに切り替わる |
| 食品削除 | 「削除」ボタンで削除。関連する食事記録エントリも CASCADE 削除される |
| 検索フィルター | 一覧右上の検索ボックスでインクリメンタル絞り込み |

**対応単位:** g / ml / 個 / 本 / 杯 / 切れ / 枚 / 袋 / 口 / 粒 / パック / 食分

---

### グラフタブ (`src/ui/graph.rs`)
体重の推移を折れ線グラフで表示する画面。外部ライブラリを使わず egui の Painter API で手描きしています。

| 機能 | 説明 |
|---|---|
| 折れ線グラフ | 体重履歴を時系列で表示。データポイント・グリッド線・Y 軸ラベル付き |
| 期間切り替え | 30日 / 60日 / 90日 のボタンで表示範囲を切り替え |
| 統計表示 | 最新・平均・最低・最高の体重を数値で表示 |

---

### レポートタブ (`src/ui/report.rs`)
週次の食事データを集計し、AI による評価コメントを生成する画面。

| 機能 | 説明 |
|---|---|
| 週次テーブル | 直近 7 日間の日別 kcal / P / F / C を表示。目標超過日は赤字 |
| 平均表示 | 7 日間の平均カロリーと目標との差を色で強調表示 |
| AI コーチ | 「レポートを生成」ボタンで Gemini API を呼び出し、300〜400字の評価コメントを生成 |
| 設定 | 1 日の目標摂取カロリーを設定して保存 |
| 日曜日通知 | 日曜日に起動した場合、自動でこのタブが開きレポート促進バナーを表示 |

## ファイル構成

```
calorie-app/
├── Cargo.toml              # 依存クレート定義
├── schema.sql              # SQLite スキーマ定義（コンパイル時に埋め込み）
├── .env                    # 環境変数ファイル（要作成、Git 管理外）
├── app.db                  # SQLite データベース（起動時に自動生成、Git 管理外）
└── src/
    ├── main.rs             # エントリポイント・フォント/スタイルセットアップ
    ├── domain.rs           # 型定義（ビジネスロジック・I/O なし）
    ├── app.rs              # App 状態・ビジネスロジック・egui ループ
    ├── llm.rs              # Gemini API 呼び出し
    ├── db/
    │   ├── mod.rs          # SQLite アクセス層（Db 構造体）
    │   ├── schema.rs       # スキーマ初期化・バージョン管理
    │   └── seed.rs         # シードデータ投入（現在は空）
    └── ui/
        ├── mod.rs          # タブナビ・共有ウィジェット・カラーパレット
        ├── home.rs         # 今日スクリーン
        ├── foods.rs        # 食品管理スクリーン
        ├── graph.rs        # グラフスクリーン
        └── report.rs       # レポートスクリーン
```

### 各モジュールの責務

| モジュール | 責務 |
|---|---|
| `main.rs` | 起動シーケンス（.env 読み込み → DB オープン → ウィンドウ起動）、フォント/テーマ設定 |
| `domain.rs` | `FoodItem` / `LoggedItem` / `Summary` / `Slot` / `Unit` などの純粋なデータ型。I/O なし |
| `app.rs` | `App` 構造体（全画面の状態を保持）、体重/食事/食品マスタの保存処理、LLM レポートの非同期起動 |
| `llm.rs` | Gemini API への HTTP リクエスト（ureq）とレスポンス解析 |
| `db/mod.rs` | SQL の実行と Rust 型への変換のみを担う薄いラッパー |
| `db/schema.rs` | スキーマバージョン管理。`DB_VERSION` を上げると全テーブルを再作成 |
| `ui/mod.rs` | タブバー・トップバーの描画、`card`/`nutrient_bar` などの共有ウィジェット、カラー定数 |
| `ui/home.rs` | 今日スクリーンの描画（体重カード・サマリカード・食事記録カード・ログ一覧） |
| `ui/foods.rs` | 食品管理スクリーンの描画（追加/編集フォーム・食品一覧） |
| `ui/graph.rs` | グラフスクリーンの描画（折れ線グラフ・統計カード） |
| `ui/report.rs` | レポートスクリーンの描画（週次テーブル・AI コーチカード・設定カード） |

## データベーススキーマ

```sql
-- アプリ設定の KV ストア
settings        (key TEXT PK, value TEXT)

-- 食品マスタ（ユーザーが登録）
food_item       (id, name UNIQUE, unit, kcal_per_unit, p_per_unit, f_per_unit, c_per_unit)

-- 食事記録（日付×スロットで1行）
meal_log        (id, date, slot, UNIQUE(date, slot))

-- 食事記録の食品エントリ（meal_log に紐付く）
meal_log_item   (id, meal_log_id FK, food_item_id FK, amount)
                -- 両 FK とも ON DELETE CASCADE

-- 体重記録（日付が主キー）
weight_log      (date PK, weight_kg)
```

## セットアップ

### 前提条件

- Rust 1.70 以上（`rustup` でインストール推奨）

### 1. リポジトリのクローン

```bash
git clone <リポジトリURL>
cd calorie-app
```

### 2. 環境変数の設定

プロジェクトルートに `.env` ファイルを作成します。

```bash
# .env
GEMINI_API_KEY=your_gemini_api_key_here
```

> AI コーチ機能を使わない場合はこのファイルの作成は不要です。

Gemini API キーは [Google AI Studio](https://aistudio.google.com/) から取得できます。

### 3. ビルドと起動

```bash
cargo run
```

初回起動時に `app.db` が自動生成されます。

## 操作方法

### 食品の登録（最初にやること）

1. **「食品」タブ** を開く
2. フォームに食品名・単位・1単位あたりの栄養素を入力
3. **「登録」** ボタンをクリック

### 食事の記録

1. **「今日」タブ** を開く
2. 朝食 / 昼食 / 夕食 / 間食のボタンでスロットを選択
3. 食品名の検索ボックスに文字を入力して候補を絞り込む
4. 候補チップをクリックして食品を選択
5. 量を入力して **「追加」** ボタンをクリック
6. 削除したい場合は食品行の **「×」** ボタン

### 体重の記録

1. **「今日」タブ** の体重欄に kg を入力
2. **「保存」** ボタンまたは **Enter キー** で保存

### グラフの確認

1. **「グラフ」タブ** を開く
2. **30日 / 60日 / 90日** ボタンで表示期間を切り替え

### 週次レポートの生成

1. **「レポート」タブ** を開く（日曜日は自動で開く）
2. 直近 7 日間の食事記録が表示される
3. **「レポートを生成」** ボタンをクリック（`GEMINI_API_KEY` が必要）
4. 数秒待つと AI コーチのコメントが表示される

### 目標カロリーの設定

1. **「レポート」タブ** の下部「設定」カード
2. 目標摂取カロリーを入力
3. **「設定を保存」** をクリック

## データの保存場所

| ファイル | 内容 |
|---|---|
| `app.db` | すべての食事・体重・設定データ（SQLite） |
| `.env` | Gemini API キー |

どちらも `.gitignore` に登録済みで Git 管理外です。

---

## 注意事項

- **スキーママイグレーション:** `src/db/schema.rs` の `DB_VERSION` を変更すると、起動時に全テーブルが削除・再作成されます（既存データは失われます）。
- **日本語フォント:** macOS の Hiragino フォントを自動検索します。見つからない場合は egui のデフォルトフォントにフォールバックします。
- **LLM 呼び出しはブロッキング:** Gemini API の呼び出しは別スレッドで実行されるため、生成中も UI は操作可能です。
